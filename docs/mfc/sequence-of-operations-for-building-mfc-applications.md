# <a name="sequence-of-operations-for-building-mfc-applications"></a>MFC 응용 프로그램을 빌드하는 작업 시퀀스

다음 표에서 MFC 응용 프로그램을 개발 하는 경우 일반적으로 따르는 일반적인 순서를 보여 줍니다.

### <a name="sequence-for-building-an-application-with-the-framework"></a>프레임 워크를 사용 하 여 응용 프로그램을 빌드하기 위한 시퀀스

|작업|너 해|프레임 워크|
|----------|------------|------------------------|
|기본 응용 프로그램을 만듭니다.|실행 합니다 [MFC 응용 프로그램 마법사](../mfc/reference/mfc-application-wizard.md)합니다. 옵션 페이지에서 원하는 옵션을 지정 합니다. COM 구성 요소, 컨테이너 또는 두 작업 모두 응용 프로그램을 만드는 옵션 포함 추가 자동화; 응용 프로그램을 데이터베이스에서 인식 합니다.|MFC 응용 프로그램 마법사에 응용 프로그램, 문서, 뷰 및 프레임 창;에 대 한 소스 파일을 포함 하 여 기본 응용 프로그램에 대 한 파일을 만듭니다. 리소스 파일입니다. 프로젝트 파일 및 다른 모든 맞게 사양입니다.|
|뿐만 아니라 사용자 고유의 코드 줄을 추가 하지 않고 프레임 워크 및 MFC 응용 프로그램 마법사를 제공 하는 항목을 참조 하세요.|기본 응용 프로그램을 빌드하고 Visual c + +에서 실행 합니다.|실행 중인 기본 응용 프로그램에는 많은 표준 파생 **파일**, **편집**, **뷰**, 및 **도움말** 프레임 워크에서 메뉴 명령을 합니다. MDI 응용 프로그램에 대 한 모든 기능을 갖춘 Windows 메뉴를 표시 및 프레임 워크 만들기, 정렬 및 MDI 자식 창 소멸을 관리 합니다.|
|응용 프로그램의 사용자 인터페이스를 생성 합니다.|Visual c + + [리소스 편집기](../windows/resource-editors.md) 시각적으로 응용 프로그램의 사용자 인터페이스를 편집 하려면:<br /><br /> -메뉴를 만듭니다.<br />액셀러레이터 키를 정의 합니다.<br />-대화 상자를 만듭니다.<br />-만들기 및 비트맵, 아이콘 및 커서를 편집 합니다.<br />-MFC 응용 프로그램 마법사에서 생성 하는 도구 모음에서 편집 합니다.<br />-만들기 및 기타 리소스를 편집 합니다.<br /><br /> 또한 대화 상자 편집기에서 대화 상자를 테스트할 수 있습니다.|MFC 응용 프로그램 마법사로 만든 기본 리소스 파일에 필요한 리소스를 많이 제공 합니다. Visual c + +를 사용 하 여 기존 리소스를 편집 하 고 쉽고 시각적으로 새 리소스를 추가할 수 있습니다.|
|메뉴 처리기 함수에 매핑됩니다.|사용 하 여를 **이벤트** 단추를 [속성 창](/visualstudio/ide/reference/properties-window) 메뉴 및 액셀러레이터 코드의 처리기 함수에 연결 합니다.|속성 창 소스 파일 지정 하 고 많은 수동 코딩 작업을 관리에 빈 함수 템플릿과 메시지 맵 항목을 삽입 합니다.|
|처리기 코드를 작성 합니다.|클래스 뷰를 사용 하 여 소스 코드 편집기에서 코드로 직접 이동 합니다. 처리기 함수에 대 한 코드를 입력 합니다. 클래스 뷰를 사용 하 여 및 프로젝트에 코드를 추가 하는 마법사에 대 한 자세한 내용은 참조 하세요 [코드 마법사로 기능 추가](../ide/adding-functionality-with-code-wizards-cpp.md)합니다.|클래스 뷰 편집기를 엽니다., 빈 함수 템플릿에 스크롤하고 커서를 배치 합니다.|
|도구 모음 단추를 명령에 매핑하십시오.|단추를 적절 한 명령 ID를 할당 하 여 하면 도구 모음의 각 단추는 액셀러레이터 키 또는 메뉴 명령에 매핑|프레임 워크에는 그리기, 활성화, 비활성화, 검사 및 도구 모음 단추의 시각적 측면이 제어 합니다.|
|처리기 함수를 테스트 합니다.|프로그램을 다시 빌드하고 처리기 올바르게 작동 하는지 테스트 하려면 기본 제공 디버깅 도구를 사용 합니다.|단계 또는 처리기 라고 하는 방법을 보려면 코드를 통해 추적 수 있습니다. 처리기 코드를 작성 하는 경우 처리기는 명령을 수행 합니다. 프레임 워크 처리 되지 않는 도구 모음 단추 및 메뉴 항목에 자동으로 비활성화 됩니다.|
|추가 [대화 상자](../mfc/dialog-boxes.md)합니다.|대화 상자 편집기를 사용 하 여 대화 상자 템플릿 리소스를 디자인 합니다. 그런 다음 대화 상자 클래스 및 대화 상자를 처리 하는 코드를 만듭니다.|프레임 워크는 대화 상자를 관리 하 고 쉽게 사용자가 입력 한 정보를 검색할 수 있도록 합니다.|
|초기화 하 고 유효성 검사 대화 상자 데이터를 검색 합니다.|대화 상자의 컨트롤이 초기화 하 고 확인 하는 하는 방법을 정의할 수 있습니다. Visual Studio를 사용 하 여 대화 상자 클래스에 멤버 변수를 추가 하 여 대화 상자 컨트롤에 매핑합니다. 사용자가 데이터 각 컨트롤에 적용할 유효성 검사 규칙을 지정 합니다. 원하는 경우 사용자 고유의 사용자 지정 유효성 검사를 제공 합니다.|프레임 워크는 대화 상자 초기화 및 유효성 검사를 관리합니다. 사용자가 잘못 된 정보를 입력 하는 경우 프레임 워크는 메시지 상자를 표시 하 고 사용자는 데이터를 다시 입력할 수 있습니다.|
|추가 클래스를 만듭니다.|추가 문서, 뷰 및 MFC 응용 프로그램 마법사에서 자동으로 만든 것 이상의 프레임 창 클래스를 만들려면 클래스 뷰를 사용 합니다. 추가 데이터베이스 레코드 집합 클래스, 대화 상자 클래스 및 등을 만들 수 있습니다. (클래스 뷰를 사용 하 여 만들 수 있습니다 MFC 클래스에서 파생 되지 않은 클래스.)|클래스 뷰 소스 파일에 이러한 클래스를 추가 하 고 처리 하는 명령에 해당 연결을 정의할 수 있습니다.|
|응용 프로그램에 즉시 사용할 구성 요소를 추가 합니다.|사용 하 여는 `New Item dialog box` 다양 한 항목을 추가 합니다.|이러한 항목은 쉽게 응용 프로그램에 통합 하 고 많은 양의 작업을 저장 합니다.|
|문서 클래스를 구현 합니다.|응용 프로그램 관련 문서 클래스 또는 클래스를 구현 합니다. 데이터 구조를 저장할 멤버 변수를 추가 합니다. 데이터에 대 한 인터페이스를 제공 하는 멤버 함수를 추가 합니다.|프레임 워크에는 문서 데이터 파일과 상호 작용 하는 방법을 이미 알고 있습니다. 이 수 문서 파일을 닫습니다, 그리고 읽기 및 문서의 데이터를 쓸 열고 다른 사용자 인터페이스를 처리 합니다. 문서의 데이터를 조작 하는 방법에 집중할 수 있습니다.|
|열기, 저장을 구현 하 고 명령으로 저장 합니다.|문서에 대 한 코드를 작성할 `Serialize` 멤버 함수입니다.|프레임 워크에 대 한 대화 상자를 표시 합니다 **열려**, **저장**, 및 **이름으로 저장** 명령을 **파일** 메뉴. 기록 및 읽기에 지정 된 데이터 형식을 사용 하 여 문서를 다시 여 `Serialize` 멤버 함수입니다.|
|뷰 클래스를 구현 합니다.|문서에 해당 하는 하나 이상의 뷰 클래스를 구현 합니다. 클래스 뷰를 사용 하 여 사용자 인터페이스에 매핑되는 보기의 멤버 함수를 구현 합니다. 다양 한 [CView](../mfc/reference/cview-class.md)-파생된 클래스는 포함 하 여 사용 가능한 [CListView](../mfc/reference/clistview-class.md) 하 고 [CTreeView](../mfc/reference/ctreeview-class.md)합니다.|프레임 워크는 대부분의 문서와 뷰 간의 관계를 관리합니다. 보기의 멤버 함수 화면이 나 인쇄 된 페이지에서 해당 이미지를 렌더링 하 고 명령을 편집 하는 사용자에 대 한 응답에서 문서 데이터 구조를 업데이트 하는 뷰의 문서에 액세스 합니다.|
|기본 인쇄가 강화.|여러 페이지로 된 인쇄를 지원 해야 하는 경우 보기 멤버 함수를 재정의 합니다.|프레임 워크 지원 합니다 **인쇄**를 **페이지 설정**, 및 **인쇄 미리 보기** 명령을 합니다 **파일** 메뉴. 여러 페이지에 문서를 중단 하는 방법을 알려야 합니다.|
|스크롤을 추가 합니다.|스크롤을 지원 해야 하는 경우 뷰 클래스 또는 클래스에서 파생 [CScrollView](../mfc/reference/cscrollview-class.md)합니다.|뷰는 뷰 창에는 개체가 너무 작아서 때 자동으로 스크롤 막대를 추가 합니다.|
|폼 뷰를 만듭니다.|대화 상자 템플릿 리소스에서 보기를 기본 하려는 뷰 클래스 또는 클래스에서 파생 [CFormView](../mfc/reference/cformview-class.md)합니다.|뷰는 컨트롤을 표시 하는 대화 상자 템플릿 리소스를 사용 합니다. 사용자 컨트롤 보기에 컨트롤에서 탭 수입니다.|
|데이터베이스 폼을 만듭니다.|폼 기반 데이터 액세스 응용 프로그램을 원한다 면 보기에서 파생 [CRecordView](../mfc/reference/crecordview-class.md) (ODBC 프로그래밍)에 대 한 합니다.|뷰는 폼 보기 처럼 작동 하지만 해당 컨트롤의 필드에 연결 되어는 [CRecordset](../mfc/reference/crecordset-class.md) 데이터베이스 테이블을 나타내는 개체입니다. MFC를 컨트롤과 레코드 집합 간에 데이터를 이동합니다.|
|간단한 텍스트 편집기를 만듭니다.|에 뷰를 간단한 텍스트 편집기를 원한다 면 보기 클래스 또는 클래스에서 파생 [CEditView](../mfc/reference/ceditview-class.md) 하거나 [CRichEditView](../mfc/reference/cricheditview-class.md)합니다.|함수, 클립보드 지원 및 입/출력 파일을 편집 보기를 제공 합니다. `CRichEditView` 스타일이 적용 된 텍스트를 제공합니다.|
|분할자 창을 추가 합니다.|창을 분할을 지원 하려는 경우 추가 [CSplitterWnd](../mfc/reference/csplitterwnd-class.md) SDI 프레임 창 또는 MDI 자식 창에 개체 및 창에 후크 [OnCreateClient](../mfc/reference/cframewnd-class.md#oncreateclient) 멤버 함수입니다.|프레임 워크는 스크롤 막대 옆에 있는 분할 상자 컨트롤을 제공 하 고 여러 개의 창으로 보기 분할을 관리 합니다. 사용자가 창을 분할을 하는 경우 프레임 워크를 만들고 문서에 추가 뷰 개체를 연결 합니다.|
|빌드, 테스트 및 응용 프로그램을 디버그 합니다.|빌드, 테스트 및 응용 프로그램을 디버깅 하려면 Visual c + +의 기능을 사용 합니다.|Visual c + +를 사용 하면 컴파일, 링크 및 기타 옵션을 조정할 수 있습니다. 또한 소스 코드와 클래스 구조를 탐색할 수 있습니다.|
